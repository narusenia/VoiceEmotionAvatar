# VEA 実装計画書

## コミット計画（論理単位）

各ステップを1コミットとし、Conventional Commits規約に従う。

---

### Step 1: `chore: プロジェクト初期化`

- `pyproject.toml` 作成（uv用、依存関係定義）
- `.gitignore` 作成
- ディレクトリ構造作成
- README.md（最小限の起動手順）

```
VoiceEmotionAvatar/
├── docs/
│   ├── requirements.md
│   └── implementation-plan.md
├── src/
│   └── vea/
│       ├── __init__.py
│       ├── main.py          # エントリーポイント
│       ├── audio.py          # 音声キャプチャ
│       ├── emotion.py        # 感情認識（emotion2vec）
│       ├── smoother.py       # ヒステリシス付き補間
│       ├── osc_sender.py     # OSC送信
│       ├── gui.py            # Dear PyGui GUI
│       └── config.py         # 設定管理
├── pyproject.toml
├── .gitignore
└── README.md
```

### Step 2: `feat: 設定管理モジュール`

- `config.py`: YAML/JSON設定ファイルの読み書き
- デフォルト設定の定義
  - マイクデバイス（None = システムデフォルト）
  - OSC送信先（127.0.0.1:9000）
  - 分析間隔（250ms）
  - 補間速度・閾値パラメータ
  - 感情パラメータ名プレフィックス

### Step 3: `feat: 音声キャプチャモジュール`

- `audio.py`: sounddeviceによるマイク入力
  - デバイス列挙
  - リングバッファによるストリーミング取得
  - 250ms分の音声チャンクを切り出す
  - マイク切断検出 + 自動リトライ

### Step 4: `feat: 感情認識モジュール`

- `emotion.py`: emotion2vecによる推論
  - モデルのロード（初回起動時に自動ダウンロード）
  - 音声チャンク → 5感情の確率を出力
  - GPU(CUDA)での推論
  - emotion2vecの出力ラベルを VEA の5感情にマッピング

### Step 5: `feat: スムージングモジュール`

- `smoother.py`: ヒステリシス付きLerp
  - 現在値と目標値のLerp補間（補間速度は設定可能）
  - 感情切替の閾値（ヒステリシス）
  - チャタリング防止ロジック

### Step 6: `feat: OSC送信モジュール`

- `osc_sender.py`: python-oscによるOSC送信
  - UDPクライアント生成
  - 5つのFloatパラメータを送信
  - 送信先IP・ポートの動的変更対応

### Step 7: `feat: GUIモジュール`

- `gui.py`: Dear PyGuiによるUI
  - メインウィンドウ
  - マイクデバイスドロップダウン
  - 感情リアルタイムモニター（水平バー × 5感情）
  - 感度・閾値スライダー
  - 開始/停止ボタン
  - 高度な設定セクション（折りたたみ）
    - OSC IP・ポート入力
  - エラー通知ポップアップ

### Step 8: `feat: メインループとアプリケーション統合`

- `main.py`: 全モジュールの統合
  - 起動シーケンス（設定読込 → モデルロード → GUI起動）
  - メインループ（音声取得 → 推論 → スムージング → OSC送信 → GUI更新）
  - スレッド設計:
    - メインスレッド: Dear PyGui描画ループ
    - ワーカースレッド: 音声取得 → 推論 → OSC送信パイプライン
  - シャットダウン処理

### Step 9: `test: 動作確認と調整`

- 実際にVRChatで動作確認
- パラメータチューニング（閾値・補間速度のデフォルト値調整）
- 必要に応じてバグ修正

## 依存ライブラリ

| ライブラリ | 用途 |

## リスクと対策

| リスク | 対策 |
|---|---|
| emotion2vecの出力ラベルがVEAの5感情と一致しない | マッピングテーブルで変換。モデル調査後に確定 |
| GPU VRAMの競合（VRChat + emotion2vec） | emotion2vecは小型モデル（~300MB）なので問題なし |
| 音声が無い時のノイズ推論 | 音量閾値を設けて無音時はNeutralにフォールバック |
| Dear PyGuiとワーカースレッドの競合 | キューまたはスレッドセーフな共有変数で通信 |

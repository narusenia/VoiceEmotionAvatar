# VoiceEmotionAvatar 使い方ガイド

声の感情をAIで判別して、VRChatのアバターの表情をリアルタイムに動かすツールです。
笑えば笑顔に、怒れば怒り顔に──マイクに向かって喋るだけで表情が変わります。

このガイドは「初めて使う人」が最後まで設定して動かせることを目標にした手順書です。
各設定の詳しいリファレンスは [README.md](README.md) を参照してください。

---

## 全体の流れ

セットアップは大きく4ステップです。

1. **アプリを導入する**（PC側）… `start.bat` をダブルクリックするだけ
2. **アバターに表情を仕込む**（Unity側）… 表情アニメを作ってアバターに組み込む
3. **VRChatでOSCを有効化する**
4. **アプリを起動して喋る**

> 2 のUnity作業は **アバター1体につき最初の1回だけ** です。一度設定すれば、以降は 1 → 3 → 4 だけで使えます。

---

## 必要なもの

### PC環境
- Windows 10 / 11
- **NVIDIA GPU**（CUDA対応、RTX3060以上を推奨）
- インターネット接続（初回のダウンロードに必要）

> Python や uv の事前インストールは不要です。初回起動時に自動で用意されます。

### アバター改変環境（STEP 2 で使用）
- Unity（アバターの対応バージョン）
- **VRChat SDK3 - Avatars**
- **[Modular Avatar](https://modular-avatar.nadena.dev/)**（推奨。非破壊でセットアップできます）

---

## STEP 1: アプリの導入と起動

1. このプロジェクト一式を任意のフォルダに置きます。
2. **`start.bat` をダブルクリック** します。
3. 初回は以下が自動でダウンロードされます（合計で数GB・時間がかかります）。
   - uv（環境マネージャ）
   - Python 3.12 + PyTorch ほか依存ライブラリ（約2.5GB）
   - emotion2vec モデル（初回に「Start」した時、数百MB）
4. しばらくするとGUIウィンドウが開きます。

2回目以降は `start.bat` を押せばすぐ起動します。

> うまく動かないときは末尾の [トラブルシューティング](#トラブルシューティング) を参照してください。

---

## STEP 2: アバター側のセットアップ（Unity）

声の感情に対応する**表情アニメーション**をアバターに用意し、それをアバターに組み込みます。

### 2-1. エディタツールを導入する

**方法A: unitypackage（推奨）**
[Releases](https://github.com/narusenia/VoiceEmotionAvatar/releases) から `VEA-vX.Y.Z.unitypackage` をダウンロードし、Unity にドラッグ＆ドロップでインポートします。`Assets/VEA/Editor/` に展開されます。

**方法B: 手動コピー**
`unity/Assets/VEA/Editor/` の中身を、自分のUnityプロジェクトの `Assets/VEA/Editor/` にコピーします。

> 前提として VRChat SDK3 (Avatars) と Modular Avatar がプロジェクトに入っている必要があります。

導入が成功すると、メニューバーに **Tools → VEA** が増えます。

### 2-2. 表情アニメを作る（BlendShape Editor）

メニューから **Tools → VEA → BlendShape Editor** を開きます。

1. **Avatar** 欄に、シーン上のアバター（VRCAvatarDescriptor）をドラッグします。
   - 顔メッシュ（Face Mesh）は自動推定されます。違う場合は手動でドラッグして指定します。
2. 感情を5種類で使うか7種類で使うかを **Full Mode (7 emotions)** のチェックで切り替えます。
   - OFF（5種）= Joy / Anger / Sadness / Surprise / Neutral
   - ON（7種）= 上記 + Disgust / Fear
   - ※ ここの設定は STEP 2-3 のセットアップやアプリ側の Full Mode と揃えてください。
3. 上部の**感情タブ**（Joy, Anger…）から、設定したい感情を選びます。
4. 下の「**BlendShape を追加**」で、使いたいシェイプを探します。
   - 検索ボックスにキーワードを入れると絞り込めます（例: `smile`, `mouth`, `eye`）。
   - 候補一覧から **「+ 追加」** で表情に組み込みます。
5. 追加したシェイプは上の「**設定済み**」に並びます。スライダー（0〜100）で効き具合を調整します。不要なら **×** で削除します。
6. **▶ Preview** を押すと、シーンのアバターに今の表情がリアルタイムで反映されます。確認したら **■ Stop Preview** で元に戻ります。
7. 各感情を作り終えたら **Save All** を押します。
   - `Assets/VEA/Generated/Animations/VEA_<感情>.anim` として保存されます。
   - 1つだけ保存したいときは **Save Current** を使います。

> **コツ**: Neutral（無表情）は「すべて0」のままでも構いません。喜び・怒りなど分かりやすい感情から作ると調整しやすいです。

### 2-3. アバターに組み込む

作った表情アニメをアバターに紐づけます。**どちらか一方**を実行します。

**推奨: Modular Avatar で非破壊セットアップ**
**Tools → VEA → Setup with Modular Avatar** を開き、アバターをセットして実行します。
- 既存のFXコントローラを書き換えないため、Face-Emo など他の表情ツールと併用しやすいです。
- Expression Parameters と表情の合成（Merge Animator）を自動で構成します。

**直接セットアップ（MAを使わない場合）**
**Tools → VEA → Setup Avatar** を開き、アバターをセットして実行します。
- Expression Parameters と FXレイヤー（`VEA_Emotions`）を直接追加します。
- VEAのレイヤーは**一番最後**に配置されます（他レイヤーに上書きされないように）。

> どちらも実行時に **Expression Parameters のコスト（上限256）** をチェックします。超える場合は警告が出るので、不要なパラメータを減らしてください。

### 2-4. アップロード

通常どおりアバターをVRChatにアップロードします。

---

## STEP 3: VRChatでOSCを有効化する

1. VRChat内で **Action Menu → Options → OSC → Enabled** にします。
2. 初回有効化時は、いったん別アバターに着替えてから戻すと、OSCのパラメータファイルが再生成されて確実です。

---

## STEP 4: アプリを使う

`start.bat` でアプリを起動し、以下を操作します。

### 基本操作
| 項目 | 説明 |
|---|---|
| **Microphone** | 使用するマイクを選択 |
| **Input Gain** | マイク入力の増幅（声が小さいとき上げる） |
| **Level** | 現在の入力音量メーター |
| **Start / Stop** | 感情認識パイプラインの開始・停止 |
| **Emotion Monitor** | 各感情の確率をリアルタイム表示 |
| **Full Mode (7 emotions)** | 7感情モード。STEP 2 の設定と揃える |

**「Start」を押すと初回はモデルのダウンロードが入ります。** ダウンロード後、喋るとモニターのバーが動き、VRChatのアバター表情が変化します。

### Mode（切り替え方）
- **Instant Mode（閾値超えで即時切替）**: ONにすると、最も確率の高い感情が閾値を超えたとき、その感情だけを表示する「はっきり切替」モードになります。
  - **Instant Threshold**（0.1〜0.9, 既定0.4）: 切り替わる確率の閾値。低いと敏感、高いとはっきり喋らないと切り替わりません。
  - **Smoothing**（0.0〜1.0, 既定0.5）: 切替時のフェード。0=パッと、1=ゆっくり。
- **Smooth Mode（Instant OFF時）**: 全感情の確率をそのままブレンドする滑らかなモードです。
  - **Lerp Speed**（既定0.15）: 目標値への追従の速さ。高いほど速い。
  - **Hysteresis**（既定0.10）: 感情が切り替わるのに必要な確率差。表情のパタパタ（チャタリング）を防ぎます。
  - **Silence Threshold**（既定0.01）: この音量未満は無音とみなしNeutralに戻します。環境ノイズが多いなら上げます。
  - **Hold (sec)**（既定0.0）: 感情を切り替えた後、次に切り替わるまでの最短保持時間。

### Manual Mode
**Manual Mode** をONにすると、感情認識を使わず**各感情のスライダーを手動で動かして**表情を確認・テストできます。セットアップ確認や配信演出に便利です。

### Advanced Settings
- **OSC IP**（既定 127.0.0.1）/ **OSC Port**（既定 9000）: VRChatの受信先。通常は変更不要です。変更したら **Apply OSC Settings** を押します。

---

## 調整のコツ

| 困りごと | 対処 |
|---|---|
| 表情がパタパタ切り替わる | **Hysteresis** を上げる / **Hold** を 0.3〜0.5 秒に / **Instant Mode** を使う |
| 反応が鈍い・もっさり | **Lerp Speed** を上げる / **Instant Mode** + **Smoothing** を下げる |
| 喋っていないのに表情が動く | **Silence Threshold** を上げる |
| 声が小さくて反応しない | **Input Gain** を上げる（上げすぎると割れるので注意） |
| 特定の感情が出にくい | Instant Mode なら **Instant Threshold** を下げる |

## トラブルシューティング

**`start.bat` がエラーで止まる / ダウンロードに失敗する**
- インターネット接続を確認してください。社内プロキシなどがあると uv の取得に失敗することがあります。
- 一度失敗した場合、フォルダ内の `.uv` と `.venv` を削除してから `start.bat` を再実行すると直ることがあります。

**「Start」してもモデルのロードに失敗する / 動作が極端に遅い**
- NVIDIA GPU と最新ドライバが必要です。GPUが無い環境では実用的な速度になりません。
- 初回はモデル（数百MB）のダウンロードが入ります。完了まで待ってください。

**マイクが一覧に出ない / 認識されない**
- OSのサウンド設定でマイクが有効か、既定デバイスになっているか確認してください。
- アプリ起動中にマイクを挿し直した場合は、Microphone を選び直してください（自動再接続も試みます）。

**VRChatでアバターの表情が動かない**
- VRChat側で **OSCがEnabled** か確認（STEP 3）。一度アバターを着替えて戻すと確実です。
- アプリの **OSC IP/Port** が `127.0.0.1 / 9000`（既定）になっているか確認。
- アバターに **Expression Parameters**（`VEA_Joy` など）が入っているか、**FXレイヤーが一番最後**にあるか確認（STEP 2-3）。
- アプリの **Full Mode** と、STEP 2 で作った感情数（5 or 7）が一致しているか確認。
- まず **Manual Mode** でスライダーを動かし、表情が変わるか切り分けると原因を特定しやすいです。

## 参考情報

### OSCパラメータ一覧
アプリは各感情の確率を Float (0.0〜1.0) で送信します。

| パラメータ名 | 感情 |
|---|---|
| `VEA_Joy` | 喜び |
| `VEA_Anger` | 怒り |
| `VEA_Sadness` | 悲しみ |
| `VEA_Surprise` | 驚き |
| `VEA_Neutral` | ニュートラル |
| `VEA_Disgust` | 嫌悪（Full Modeのみ） |
| `VEA_Fear` | 恐怖（Full Modeのみ） |

### 設定ファイルの場所
GUIで変更した設定は `~/.vea/config.yaml`（`C:\Users\<ユーザー名>\.vea\config.yaml`）に自動保存されます。

### 生成物の場所
- 表情アニメ: `Assets/VEA/Generated/Animations/VEA_<感情>.anim`（Unityプロジェクト内）
